# 基于强化学习的果园无人机喷洒路径规划实验设计与结果分析

## 1. 实验目的

针对果园植保作业中无人机喷洒路径规划存在的覆盖效率低、重复喷洒率高以及障碍物规避能力不足等问题，本文基于 Aerial Autonomy Stack（AAS）原有 Gazebo 果园仿真环境，构建无人机喷洒路径规划实验平台，并将该任务建模为强化学习问题。实验目标是在保证目标树木喷洒覆盖率和障碍物安全避让的前提下，尽可能缩短无人机飞行路径，减少重复喷洒，并生成可在 AAS/Gazebo 中执行的飞行任务文件。

本实验并未重新构建独立的果园地图，而是直接复用 AAS 工程中已有的 `apple_orchard.sdf` 仿真世界。实验程序解析 SDF 文件中当前启用的模型实体，并根据树木和障碍物的位置自动构建规划网格。这样可以保证算法训练和路径评估均来自原始仿真环境，而不是脱离仿真系统的手工地图。

## 2. 实验平台与环境构建

实验平台基于 AAS 无人机自主仿真系统，包括 Gazebo Sim 三维仿真环境、PX4/ROS 2 飞控接口、任务执行节点以及地面站显示工具。果园环境来自：

```text
simulation/simulation_resources/simulation_worlds/apple_orchard.sdf
```

程序从该 SDF 文件中解析当前处于启用状态的 `<include>` 模型，并进一步解析嵌套模型文件，例如 `birch_row`、`apple_grid`、`apple` 和 `birch`。当前实验场景中，`birch_row_1` 为启用状态，而 `apple_grid_1`、`apple_grid_2` 和 `apple_grid_3` 在 SDF 文件中仍处于注释状态。因此，当前实验实际规划目标为已启用的树行，而不是额外新建的果园地图。

当前环境统计如下：

| 项目 | 数值 |
|---|---:|
| 有效目标树木数量 | 21 |
| 静态障碍物数量 | 8 |
| 网格分辨率 | 5 m |
| 起始网格位置 | (29, 7) |
| 当前最优任务航点数 | 73 |

静态障碍物由 SDF 中启用的车辆、建筑物和人物等模型解析得到。规划层将障碍物周围一定半径内的网格标记为不可通行区域，以实现高层静态避障。目标树木所在网格被标记为喷洒目标，无人机在目标附近一定喷洒半径内经过时，视为完成对应目标喷洒。

## 3. 强化学习问题建模

本文将无人机果园喷洒路径规划问题建模为马尔可夫决策过程（Markov Decision Process, MDP），记为：

```text
M = (S, A, P, R, gamma)
```

其中，`S` 为状态空间，`A` 为动作空间，`P` 为状态转移函数，`R` 为奖励函数，`gamma` 为折扣因子。

### 3.1 状态空间

为降低训练复杂度，本文没有直接将完整二维地图作为输入，而是设计了紧凑状态向量。每一步状态由 11 维连续特征组成：

```text
s_t = [
  r_t, c_t,
  r_g, c_g,
  delta_r, delta_c,
  coverage,
  b_up, b_down, b_left, b_right
]
```

其中，`r_t` 和 `c_t` 表示无人机当前网格行列位置的归一化值；`r_g` 和 `c_g` 表示最近未喷洒目标的归一化位置；`delta_r` 和 `delta_c` 表示无人机与最近目标之间的相对位置；`coverage` 表示当前喷洒覆盖率；`b_up`、`b_down`、`b_left` 和 `b_right` 表示无人机四个方向是否被障碍物或地图边界阻挡。

该状态设计兼顾了局部避障信息、目标引导信息以及全局任务进度，使强化学习算法能够在不输入完整地图矩阵的情况下完成路径规划。

### 3.2 动作空间

对于 DQN、Double DQN、Dueling DQN、Rainbow DQN lite、PPO 和 A2C 等离散动作算法，动作空间定义为四个方向移动：

| 动作编号 | 动作含义 |
|---:|---|
| 0 | 向上移动一个网格 |
| 1 | 向下移动一个网格 |
| 2 | 向左移动一个网格 |
| 3 | 向右移动一个网格 |

SAC 和 TD3 属于连续动作强化学习算法。由于当前喷洒路径规划任务本质上是离散网格移动任务，本文为 SAC 和 TD3 构造了二维连续动作包装器，将连续动作向量映射为最近的上下左右方向。该处理使 SAC 和 TD3 能够参与对比，但其动作空间与离散算法并不完全等价，因此在结果分析中将其作为辅助对比算法。

### 3.3 奖励函数

奖励函数用于引导智能体同时满足覆盖、避障、路径效率和减少重复喷洒等目标。本文奖励函数由以下部分组成：

```text
R_t =
  r_step
  + r_new
  + r_repeat
  + r_progress
  + r_collision
  + r_finish
```

其中：

- `r_step` 为每步移动惩罚，用于鼓励更短路径；
- `r_new` 为喷洒新目标奖励；
- `r_repeat` 为重复喷洒惩罚；
- `r_progress` 为接近最近未喷洒目标的进度奖励；
- `r_collision` 为碰撞或越界惩罚；
- `r_finish` 为达到目标覆盖率后的完成奖励。

当前实现中的主要奖励项为：

```text
r_step = -0.01
r_new = 5.0 * 新喷洒目标数量
r_repeat = -0.15 * 重复喷洒目标数量
r_collision = -2.0
r_finish = +50.0
```

当无人机移动到障碍物或地图边界之外时，环境记录一次规划层碰撞，并给予惩罚；当无人机覆盖全部目标树木时，当前回合结束。

## 4. 对比算法

为验证不同路径规划策略在果园喷洒任务中的表现，本文对比了传统启发式方法和多种强化学习算法。

### 4.1 启发式基线方法

1. **Orchard row**：按照果园行方向进行遍历，模拟规则化行间巡航喷洒策略。
2. **Nearest target**：每一步选择距离当前无人机最近的未覆盖目标，属于最近邻贪心策略。

这两种方法不需要训练，可作为传统路径规划基线。

### 4.2 强化学习算法

本文实现或调用了以下强化学习算法：

| 算法 | 类型 | 说明 |
|---|---|---|
| DQN | Value-based | 使用 Stable-Baselines3 DQN，在离散动作空间中学习 Q 值函数 |
| Double DQN | Value-based | 使用在线网络选择动作、目标网络估计动作价值，缓解 Q 值过估计 |
| Dueling DQN | Value-based | 将状态价值函数和动作优势函数分离建模，提高不同动作价值估计能力 |
| Rainbow DQN lite | Value-based | 融合 Double DQN、Dueling Network 和 Prioritized Replay |
| PPO | Policy gradient | 采用截断目标函数限制策略更新幅度，提高训练稳定性 |
| A2C | Actor-Critic | 同步 Advantage Actor-Critic 方法 |
| SAC | Actor-Critic | 最大熵连续控制算法，通过连续动作包装器参与对比 |
| TD3 | Actor-Critic | 双延迟深度确定性策略梯度算法，通过连续动作包装器参与对比 |
| Maskable PPO | Policy gradient | 支持动作掩码的 PPO，本环境已提供动作掩码接口，但当前缺少 `sb3-contrib` 依赖，未参与最终训练 |

需要说明的是，本文当前实现的 Rainbow DQN lite 并非完整 Rainbow DQN。完整 Rainbow 通常包含 Double DQN、Dueling Network、Prioritized Replay、Multi-step Learning、Distributional RL 和 NoisyNet 等组件。当前实验版本主要融合了 Double DQN、Dueling Network 和 Prioritized Replay，因此更严谨的表述应为“Rainbow-style DQN lite”或“简化 Rainbow DQN”。

## 5. 评价指标

为全面评价喷洒路径规划效果，本文采用以下指标。

### 5.1 覆盖率

覆盖率表示被成功喷洒的目标树木占全部目标树木的比例：

```text
Coverage = |T_sprayed| / |T| * 100%
```

其中，`T` 为目标树木集合，`T_sprayed` 为已喷洒目标集合。覆盖率越高，说明任务完成度越好。

### 5.2 路径长度

路径长度表示无人机从起点到终点经过各航点的累计距离：

```text
L = sum distance(p_i, p_{i+1})
```

其中，`p_i` 为第 `i` 个路径点。路径长度越短，说明飞行效率越高，也意味着更低的能耗和更短作业时间。

### 5.3 重复喷洒率

重复喷洒率表示重复喷洒次数占总喷洒次数的比例：

```text
Repeat = N_repeat / (N_new + N_repeat) * 100%
```

其中，`N_new` 为首次喷洒目标次数，`N_repeat` 为重复喷洒次数。重复喷洒率越低，说明喷洒资源利用率越高。

### 5.4 碰撞次数

碰撞次数表示路径进入障碍物区域或非法区域的次数。该指标越低越好。在当前实验中，各有效算法的规划层碰撞次数均为 0。

### 5.5 航点数量

航点数量表示最终路径或 mission 文件中用于控制无人机飞行的 reposition 点数量。航点数量越少，一般说明任务执行更简洁，但仍需结合覆盖率和路径长度共同分析。

## 6. 实验设置

实验使用当前 AAS 原始果园世界中启用的树行和静态障碍物。强化学习算法统一采用 100% 目标覆盖率作为任务完成条件。当前对比实验训练步数为 20000 步，随机种子为单种子设置。DQN、PPO、A2C 等算法在离散动作空间中训练；SAC 和 TD3 通过连续动作包装器映射到离散移动方向。

训练完成后，系统会将算法输出路径转换为 AAS mission YAML 文件。当前普通 DQN 任务文件为：

```text
aircraft/aircraft_resources/missions/spray_dqn.yaml
```

当前最优 Rainbow DQN lite 任务文件为：

```text
aircraft/aircraft_resources/missions/spray_rainbow_dqn.yaml
```

其中，Rainbow DQN lite 任务文件包含 73 个 reposition 航点，可用于后续在 Gazebo 中执行三维飞行验证。

## 8. 结果分析

### 8.1 Rainbow DQN lite 的综合表现最优

从综合排序看，Rainbow DQN lite 在保证 100% 覆盖率和 0 碰撞的情况下，将路径长度降低到 360 m，相比普通 DQN 的 410 m 减少了 50 m，相比 Orchard row 的 495 m 减少了 135 m。因此，若以“覆盖率优先、碰撞次数约束、路径长度最小”为主要优化目标，Rainbow DQN lite 是当前实验中表现最优的算法。

Rainbow DQN lite 表现较好的原因在于其结合了多种 DQN 改进机制。Double DQN 能够缓解 Q 值过估计问题；Dueling Network 将状态价值和动作优势分离，有利于在多个相似动作之间形成更稳定的价值判断；Prioritized Replay 则提高了高 TD 误差样本的采样概率，使智能体更快学习关键转移样本。这些机制共同提升了算法在稀疏目标覆盖任务中的学习效率。

### 8.2 PPO 具有较强稳定性

PPO 在当前实验中也取得了较好结果，覆盖率达到 100%，路径长度为 380 m，仅次于 Rainbow DQN lite。PPO 通过限制策略更新幅度，使训练过程较为稳定。实验结果表明，虽然该任务为离散路径规划任务，PPO 仍然可以学习到有效喷洒策略。

不过，PPO 的重复喷洒率为 70.4%，高于普通 DQN 和 Rainbow DQN lite。这说明 PPO 虽然路径较短，但在覆盖过程中存在较多重复喷洒现象，喷洒资源利用效率仍有提升空间。

### 8.3 普通 DQN 的重复喷洒率最低

普通 DQN 的路径长度为 410 m，略长于 Rainbow DQN lite 和 PPO，但其重复喷洒率为 43.2%，是主要有效算法中最低的。这说明普通 DQN 学到的策略在喷洒资源利用上较为节制，能够减少对已覆盖目标的重复经过。

因此，如果任务目标更强调减少药液浪费，而不是单纯追求最短路径，普通 DQN 仍具有较高应用价值。后续研究可以通过调整奖励函数中路径长度惩罚和重复喷洒惩罚的权重，在路径效率和喷洒资源利用率之间取得更优平衡。

### 8.4 Double DQN 和 Dueling DQN 改进有限

Double DQN 和 Dueling DQN 均达到了 100% 覆盖和 0 碰撞，路径长度均为 410 m，与普通 DQN 相同。Dueling DQN 的重复喷洒率为 47.5%，优于 Double DQN 的 53.3%，但仍高于普通 DQN 的 43.2%。这表明单独引入 Double DQN 或 Dueling Network 虽能保证任务完成，但在当前场景中对路径长度的提升有限。相比之下，将多个改进机制组合起来的 Rainbow DQN lite 表现更优。

### 8.5 启发式方法可完成任务但路径较长

Nearest target 和 Orchard row 均能够实现 100% 覆盖和 0 碰撞，说明传统启发式方法在规则果园环境中仍具有可行性。然而，Nearest target 的路径长度为 445 m，Orchard row 的路径长度为 495 m，均明显长于强化学习算法。尤其是 Orchard row 按行遍历的规则性较强，虽然覆盖完整，但路径冗余较多，重复喷洒率达到 74.4%。

### 8.6 SAC、TD3 和 A2C 在当前设置下效果较差

SAC 和 TD3 是连续动作强化学习算法，而本文当前路径规划环境本质上是离散网格移动任务。虽然实验中通过二维连续动作包装器将连续动作映射为离散方向，但这种映射会削弱连续控制算法的优势。结果显示，SAC 覆盖率仅为 33.3%，TD3 覆盖率仅为 4.8%，均未能完成喷洒任务。A2C 在当前训练步数和奖励设置下也未收敛，覆盖率为 0。

因此，在当前高层网格路径规划场景中，基于离散动作空间的 DQN 系列和 PPO 更适合该任务；SAC 和 TD3 更适合后续扩展到连续速度、航向角或真实控制量输出的低层控制任务。

## 9. 最优算法任务生成与仿真执行

实验完成后，系统将最优路径转换为 AAS mission YAML 文件。当前 Rainbow DQN lite 输出的任务文件为：

```text
aircraft/aircraft_resources/missions/spray_rainbow_dqn.yaml
```

该文件包含起飞、等待、速度设置、路径航点和降落等任务步骤，其中 reposition 航点数量为 73。该任务文件可通过 AAS mission node 在 Gazebo 中执行。执行时，Gazebo Sim 窗口可显示无人机在果园环境中的三维飞行过程，QGroundControl 可显示无人机状态、轨迹和遥测信息，ROS 2 topic 可输出实时位置数据。

需要注意的是，本文当前实验结果主要为规划层验证和任务文件生成结果。完整的三维飞行验证需要进一步启动 AAS/Gazebo 仿真并执行生成的 mission 文件。

## 10. 实验局限性

尽管当前实验结果表明 Rainbow DQN lite 在果园喷洒路径规划任务中具有较好性能，但仍存在以下局限：

1. **单种子实验限制**：当前结果基于单一随机种子和 20000 步训练。严格论文实验应进一步进行多随机种子重复实验，并报告均值和标准差。
2. **静态障碍假设**：当前障碍物来自 SDF 静态模型，避障属于高层静态避障，并非基于实时视觉或雷达感知的动态避障。
3. **喷洒物理简化**：喷洒效果以目标覆盖率进行抽象建模，尚未模拟药液雾化、风场漂移和真实沉积分布。
4. **Rainbow DQN lite 非完整 Rainbow**：当前实现不包含 C51 分布式价值学习和 NoisyNet 探索机制，应在论文中明确表述为简化版本。
5. **SAC/TD3 对比不完全公平**：SAC 和 TD3 通过连续动作包装器映射到网格移动，不能完全发挥连续控制算法优势，因此其结果主要作为辅助参考。
6. **当前果园目标有限**：当前 SDF 中仅启用了 `birch_row_1`，目标数为 21。若后续启用 `apple_grid_1/2/3`，需要重新训练和评估算法。

## 11. 结论

本文基于 AAS 原有 Gazebo 果园环境，构建了无人机喷洒路径规划强化学习实验平台。通过解析原始 SDF 文件，自动提取目标树木和静态障碍物，并构建网格化喷洒规划环境。实验对比了 Orchard row、Nearest target、DQN、Double DQN、Dueling DQN、Rainbow DQN lite、PPO、A2C、SAC 和 TD3 等算法。

实验结果表明，在当前果园场景中，Rainbow DQN lite 在保证 100% 覆盖率和 0 碰撞的同时获得最短路径，路径长度为 360 m，优于普通 DQN、PPO 以及传统启发式路径规划方法。因此，Rainbow DQN lite 更适合作为当前无人机果园喷洒路径规划系统的主算法。普通 DQN 虽然路径较长，但重复喷洒率最低，说明其在喷洒资源利用方面具有优势。后续研究可进一步通过多种子实验、完整 Rainbow 组件、动态障碍感知和真实喷洒物理建模提升系统可靠性与工程应用价值。

## 12. 可直接用于论文的摘要式表述

本文设计了一种基于强化学习的果园无人机喷洒路径规划方法。首先，系统复用 AAS 原有 Gazebo 果园仿真环境，并从 SDF 文件中自动解析目标树木和静态障碍物，构建网格化喷洒规划环境。然后，将无人机喷洒任务建模为马尔可夫决策过程，设计包含位置、目标方向、覆盖率和局部障碍信息的状态空间，以及面向目标覆盖、路径缩短、重复喷洒抑制和障碍规避的奖励函数。最后，对 DQN、Double DQN、Dueling DQN、Rainbow DQN lite、PPO、A2C、SAC、TD3 以及传统启发式方法进行对比实验。结果表明，Rainbow DQN lite 在当前果园场景中取得最佳综合性能，在 0 碰撞条件下实现 100% 目标覆盖，路径长度为 360 m，相比普通 DQN 缩短 50 m，相比行遍历基线缩短 135 m。实验验证了强化学习方法在果园无人机喷洒路径规划中的可行性和优势。
